## Technical Notes

**Template Structure:**
```markdown
# {Title}

> {Brief one-line description}

**Audience:** {target audience}

---

## Overview

{2-3 paragraph introduction explaining what this documentation covers}

---

## {Section 1 Title}

<!-- vdoc:section src/path/to/file.ts -->

{Content for this section}

### {Subsection if needed}

{Details}

<!-- vdoc:end-section -->

---

## {Section 2 Title}

<!-- vdoc:section src/path/to/other.ts -->

{Content}

<!-- vdoc:end-section -->

---

## Diagram: {Diagram Title}

{Caption explaining what the diagram shows}

```mermaid
{diagram code}
```

---

## Related Documentation

- [Link to related doc](./related.md)
- [Another related doc](./another.md)

---

*Generated by vdoc v2.0.0 • Last updated: {timestamp}*
```

**Section Markers:**
The `<!-- vdoc:section path -->` comments serve two purposes:
1. Help AI identify which source file a section documents
2. Enable precise updates without affecting other sections

**Template Variants by Doc Type:**

**API Reference:**
- Group by endpoint or resource
- Include request/response examples
- HTTP method badges

**Component Library:**
- Props table for each component
- Usage examples
- Visual preview notes

**Data Model:**
- Entity descriptions
- Field tables
- Relationship diagrams

**Configuration Guide:**
- Environment variables table
- Example configurations
- Defaults and overrides
